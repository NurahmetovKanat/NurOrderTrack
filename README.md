## **2\. Технологический стек**

Выбор технологии обоснован спецификой рынка (более 70% устройств на Android в Казахстане), официальными рекомендациями Google и высокими требованиями к устойчивости работы приложения:

> * **Целевая платформа:** Min API 26 (Android 8.0) → Target API 36 (Android 16).  
> * **Язык программирования — Kotlin 2.1.0:** Официальный язык Android-разработки с 2019 года. Обеспечивает Null-safety, корутины и лаконичный синтаксис.  
> * **UI-фреймворк — Jetpack Compose \+ Material 3:** Декларативный интерфейс, единый стиль, поддержка hot-reload preview (BOM 2025.05.00).  
> * **Архитектурный паттерн — MVVM \+ Repository:** Разделение UI и бизнес-логики, применение StateFlow, слабая связность компонентов.  
> * **Облачные сервисы — Firebase Suite:** Firestore (облачная БД реального времени), Authentication, FCM (push-уведомления) — Zero-backend подход (BOM 33.14.0).  
> * **Асинхронность — Kotlin Coroutines \+ Flow v1.10.2:** Фоновые операции без блокировки интерфейса и реактивные потоки данных из Firestore.  
> * **Отчёты и Навигация:** Apache POI 5.3.0 (генерация XLSX с форматированием ячеек) и Navigation Compose 2.9.0 (type-safe переходы).

## **3\. Архитектура приложения**

Система построена на принципах слабой связности слоёв, использования реактивных потоков (StateFlow / Flow), поддержки локального кэширования (Firestore Persistence) и двойной проверки прав доступа.

### **3.1. Трёхслойная структура**

> 1. **Presentation Layer (Слой представления):**  
   * Composable-функции для декларативного UI (Jetpack Compose \+ Material 3).  
   * ViewModel с использованием StateFlow и collectAsStateWithLifecycle для автоматического обновления экранов.  
   * NavHost \+ Screen для обеспечения type-safe навигации и формирования маршрутов с учётом ролей пользователей.  
> 2. **Data Layer (Слой данных / Repository Pattern):**  
   * Repository — единый шлюз доступа к данным, изолирующий бизнес-логику от источника хранения.  
   * Kotlin Data-классы: Order, Client, Product, User, OrderItem, OrderLog.  
   * Kotlin Coroutines \+ Flow для выполнения фоновых операций и получения реактивных потоков Firestore.  
> 3. **Infrastructure Layer (Слой инфраструктуры):**  
   * Firebase Firestore — облачная NoSQL СУБД с поддержкой локального кэша (Persistence).  
   * Firebase Authentication — сессионная авторизация, bcrypt-хэширование и управление токенами.  
   * Firebase Cloud Messaging (FCM) — доставке push-уведомлений даже при закрытом приложении.  
   * Apache POI — формирование и экспорт XLSX-отчётов.

### **3.2. Ролевая модель доступа**

> * **Администратор:** Полный доступ ко всем модулям, справочнику товаров, управлению пользователями и сквозной аналитике.  
> * **Менеджер:** Доступ ко всем заказам, клиентской базе, модулю аналитики с экспортом в Excel и push-уведомлениям.  
> * **Исполнитель:** Доступ исключительно к переданным ему заказам, смене статусов выполнения, личному профилю и push-уведомлениям.

## **4\. База данных и Жизненный цикл заказа**

В качестве СУБД используется **Firebase Firestore** (документоориентированная NoSQL БД). Ключевые свойства реализации:

> * **Синхронизация:** Обновление данных у всех активных пользователей в реальном времени.  
> * **Security Rules:** Проверка правил доступа непосредственно на сервере Firebase.  
> * **Офлайн-кэш:** Локальное сохранение данных с последующей автосинхронизацией при восстановлении сети.  
> * **Денормализация:** Поле clientName сохраняется в объекте заказа (orders) для сокращения количества запросов к БД.

### **Жизненный цикл заказа**

Изменение статусов происходят строго пошагово. Порядок переходов:  
**\[Новый\] → \[В работе\] → \[Выполнен\] → \[Закрыт\] (а также возможность перехода в статус \[Отменен\])**

> * **Новый:** Создаётся менеджером или администратором.  
> * **В работе:** Исполнитель принимает задачу к выполнению.  
> * **Выполнен:** Исполнитель завершает работу по заказу.  
> * **Закрыт:** Администратор или менеджер закрывают полностью отработанный заказ.  
> * **Отменен:** Допускается на этапе от создания заказа до подтверждения его выполнения.

## **5\. Интерфейс и Ключевые Экраны**

> * **Языки интерфейса:** Поддержка 3 языков — Казахский (KZ), Русский (RU), Английский (EN).  
> * **Темы оформления:** Адаптивная светлая и тёмная темы (Material Design 3).  
> * **Основные экраны:**  
  * *Авторизация:* Вход в систему с сессионной защитой.  
  * *Dashboard:* Сводка заказов и визуализация ключевых показателей (BI).  
  * *Список заказов:* Поиск, фильтрация по статусам и датам.  
  * *Детали заказа:* Состав товаров/услуг, финансовая сумма, фиксация истории изменений и ответственных лиц.  
  * *Справочники:* Управление клиентами, сотрудниками, товарами и услугами (CRUD \+ архивация).  
  * *Аналитика:* Графики по периодам и экспорт данных в XLSX.
